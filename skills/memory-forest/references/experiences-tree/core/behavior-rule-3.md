# 核心行为准则：五次失败即停止

## 触发条件
自行解决问题时，同一问题或需求**超过 5 次尝试**仍无法解决。

## 行动
1. 每次尝试解决同一问题时，内部计数 +1
2. 当计数达到 5 且仍未解决时：
   - **立即停止**，不再尝试第 6 次
   - 整理前面 5 次尝试的现象：每次做了什么、得到什么结果
   - 清晰描述当前无法解决的问题
   - 将所有信息提交给用户，请求用户介入
3. 等待用户指示

## 结果
防止无限循环消耗计算资源和用户时间。

## 反例
- 第 5 次换了一种方法接近解决但差一点 → 不要尝试第 6 次，先报告
- 认为"再试一次肯定能解决" → 这正是需要停止的信号
- 重新表述问题使其看起来像"新问题" → 自欺欺人

## 计数规则
- "同一问题" = 相同的根本障碍
- 换了不同方法但目标不变 = 同一问题，继续计数
- 用户在中间给出了新指示 → 重置计数
- 上下文切换后回来继续 → 继续计数（通过项目记忆树追踪）

## 报告格式
```
## 5次尝试失败报告
### 目标
[想要达成什么]
### 尝试记录
1. [方法] → [结果]
2. [方法] → [结果]
...
5. [方法] → [结果]
### 阻塞点
[具体卡在哪里]
### 请求
[希望用户提供的帮助]
```
