# 文档规范

## 核心原则
- **AI 可解析优先**：所有文档结构清晰，便于 Claude Code 快速定位信息
- **可执行优于可阅读**：规范必须包含具体可操作的检查项，而非模糊原则
- **唯一真相源**：同一信息只存一处，引用而非复制

## 文件命名
- 记忆文件：`kebab-case.md`（如 `dev-standards.md`）
- 项目代码文档：`模块名.md` 或 `README.md`
- 不使用空格、下划线混用、中文文件名

## 格式规范
- 所有规范文档使用 Markdown
- 必须包含 YAML frontmatter（name, description, type）
- type 取值：`user | feedback | project | reference`
- 层级标题用 `# ## ### ####`，不超过4级
- 列表用 `-`，有序步骤用 `1.`

## 结构模板
```markdown
---
name: 规范名称
description: 一句话描述规范用途
type: project
---

# 标题

## 核心原则
（3-5条不可妥协的原则）

## 规则清单
（具体可检查的规则，每条含触发条件和处理方式）

## 禁止项
（明确不能做的事情）
```

## 语言
- 规范文档用中文编写（面向中文用户）
- 代码注释用英文（代码通用性）
- 代码标识符用英文（驼峰或蛇形命名）

## 维护规则
- 规范冲突时，优先级：安全 > 功能正确 > 性能 > 风格
- 每次发现规范不适用时，更新而非绕过
- 规范变更需记录变更理由
