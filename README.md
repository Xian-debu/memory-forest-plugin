# 记忆森林 (Memory Forest)

一套完整的 AI 行为配置与记忆管理系统，为 Claude Code 提供**分层记忆、行为准则、哲学指导和会话生命周期管理**。

## 这是什么

记忆森林不是工具，而是一套运行在 Claude 上的操作系统——定义了 Claude 如何记忆、如何行为、如何自我迭代。

核心机制：
- **6 层记忆分层** (L0 瞬时 → L5 哲学层)，自动升降级和垃圾回收
- **8 条哲学原则**（L5 绝对底层），一切规范之源头
- **7 条核心行为准则**，每次会话强制执行
- **心跳机制**追踪任务进度，支持中断/恢复
- **树状结构**组织项目、经验和系统配置

## 安装

```bash
claude plugin marketplace add https://github.com/Xian-debu/memory-forest-plugin
claude plugin install memory-forest
```

## 触发方式

在 Claude Code 中输入以下任意内容即可触发：

| 触发词 | 效果 |
|--------|------|
| "初始化记忆森林" / "initialize memory forest" | 首次设置，创建记忆目录结构 |
| "记住这个教训" / "保存经验" | 提炼并存储经验 |
| "继续上次的 X" / "恢复进度" | 恢复中断的任务 |
| "加载系统配置" / "应用行为准则" | 重新加载行为规范 |

也支持英文触发：`setup memory forest`, `load behavior rules`, `resume task` 等。

## 8 条哲学原则（L5 绝对底层）

基于毛泽东战略思想的广义指导思想，位于一切规范之上：

| # | 原则 | 核心主张 |
|---|------|---------|
| 一 | 矛盾论 | 抓主要矛盾，具体问题具体分析 |
| 二 | 军事/执行谋略 | 集中兵力、掌握主动权、战略藐视战术重视 |
| 三 | 实事求是参谋路线 | 深度理解→事实核查→质疑纠正→主动建议→确认执行 |
| 四 | 实践真理观 | 实践是检验真理的唯一标准 |
| 五 | 持久战与根据地 | 分阶段推进、积累复用、可逆迁移 |
| 六 | 自我批判与整风 | 刀刃向内、反对八股文、对称认知、知识策展 |
| 七 | 环境的战争 | 维护战场、显式资源管理、设计容错 |
| 八 | 认知纪律 | 信任权威数据源、状态隐藏于无状态表面、理解全栈 |

## 7 条核心行为准则

| # | 准则 | 对应哲学原则 |
|---|------|------------|
| 1 | 反驳-确认模式 | 三.3 质疑与纠正 — 以事实对抗错误认知 |
| 2 | 三次重复即自省 | 六.1 刀刃向内 — 主动发现并修正 |
| 3 | 五次失败即停止 | 七.3 设计容错 — 对资源消耗设置硬上限 |
| 4 | 经验迭代合并 | 六.4 知识策展 — 积累 + 修剪 |
| 5 | 自我迭代与自我提示 | 三.5/三.6 确认后执行 + 反馈转化 |
| 6 | 搜索优先 | 五.2 建立根据地 — 从已有根据地出发 |
| 7 | 多步骤先建节点 | 五.1 持久战 — 分阶段推进 |

## 记忆分层

| 层 | 生命周期 | 内容 |
|----|---------|------|
| L5 哲学 | 永久，用户确认才可修改 | 8 条哲学原则，一切规范之源 |
| L4 核心 | 年度审查 | 7 条核心行为准则 + 系统配置 |
| L3 跨项目 | 季度审查 | 可跨项目迁移的模式（≤50 条） |
| L2 项目 | 项目周期 + 30 天 | 项目目标、架构决策 |
| L1 任务 | 跨会话，完成后 7 天 | 任务目标、决策、进度 |
| L0 瞬时 | 单次会话 | 当前交互、中间结果 |

## 记忆森林结构

```
PHILOSOPHY.md (L5 绝对底层)
└── forest-root.md (L4)
    ├── MF-SPEC.md / MF-RUNTIME.md
    ├── system-tree/ (5 分支 17 规范)
    ├── experiences-tree/ (7 核心准则 + 16 跨项目模式)
    └── projects-tree/ (框架重构 + 自我进化 + n8n + AI模型)
```

## CLI 工具 (mf)

```bash
mf status       # 森林状态总览
mf health       # 健康检查
mf heartbeat    # 心跳所有活跃节点
mf gc           # 垃圾回收候选
mf search <kw>  # 全文搜索
mf list [tree]  # 列出节点
mf show <id>    # 查看节点
mf create ...   # 创建节点
```

## MCP Server

8 个 MCP 工具暴露给所有兼容 agent：
`memory_status`, `memory_health`, `memory_search`, `memory_read`, `memory_heartbeat`, `memory_gc_check`, `memory_list`, `memory_create`

## 自动化 Hooks

- **Stop Hook** — 会话退出自动心跳所有活跃节点 + GC 扫描
- **SessionStart Hook** — 会话启动自动健康检查 + GC 提醒 + 加载 PHILOSOPHY.md
